# Lab01 — ERD Design & Conceptual Modeling

## Lab Description

This lab focuses on **Entity-Relationship Diagram (ERD)** design and conceptual data modeling. Given real-world scenarios (provided as problem images), the task is to identify entities, attributes, and relationships, then draw the corresponding ERD according to database design principles.

## Topics Covered

- Identifying entities and their attributes
- Defining relationships and their cardinality (1:1, 1:N, M:N)
- Choosing primary keys and candidate keys
- Weak entities and partial keys
- ERD notation and diagram conventions
- Translating a business scenario into a conceptual data model

## Key Concepts

| Concept | Details |
|---------|---------|
| **Entity** | A real-world object/thing represented as a rectangle |
| **Attribute** | A property of an entity (simple, composite, multi-valued, derived) |
| **Relationship** | An association between two or more entities |
| **Cardinality** | Defines the numerical relationship: One-to-One, One-to-Many, Many-to-Many |
| **Weak Entity** | An entity that depends on another entity for its existence |
| **Primary Key** | The attribute(s) that uniquely identify an entity instance |
