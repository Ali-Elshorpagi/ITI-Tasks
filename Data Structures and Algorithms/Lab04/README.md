# Lab04

## Overview

This lab compares and implements classical **Sorting Algorithms**. It provides templated array permutations analyzing behavior metrics around instability, adaptivity, time complexities, and swaps logic.

## Tasks / Features

### 1. Bubble Sort (`bubble_sort.cpp`)
- Implements adjacent magnitude comparisons that repeatedly float largest/smallest items effectively simulating bubble surfaces.

### 2. Insertion Sort (`insertion_sort.cpp`)
- Designed sequentially, dragging items continuously downward left against the sorted boundaries continuously fitting pieces efficiently like sorting playing cards.

### 3. Selection Sort (`selection_sort.cpp`)
- Implements linear scans bounding minimal elements replacing bounded initial index positions reliably minimizing internal array rewriting instances.

### 4. Quick Sort (`quick_sort.cpp`)
- **Partition Algorithms**: Explores pivoting logical values (often randomized) effectively bifurcating values continuously lower/higher recursively.
- Details two different approaches for partition sorting array slices bounded within standard deviations.
- Integrates `std::mt19937` to avoid worst-case scenarios `O(N^2)` guaranteeing highly optimal split balancing via reliable randomized pivot selection.

## Key Concepts Demonstrated

- **Sorting Characteristics** — Commenting explicitly on algorithms traits like stability matching, external boundaries, algorithm scaling, and sequence mutations.
- **Time/Space Trade-offs** — Explaining comparisons logically e.g., mapping constant space structures `O(1)` alongside polynomial bounds sorting vectors `O(N^2)`.
- **Algorithm Randomization** — Modifying base assumptions via deterministic random engines (`mt19937` mapping `<random>`) enforcing median alignments explicitly dynamically.
